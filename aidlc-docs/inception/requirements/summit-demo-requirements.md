# AWS Summit デモ展示向け機能強化 - 要件定義書

## 1. インテント分析サマリー

| 項目 | 内容 |
|---|---|
| ユーザーリクエスト | 本アセットをAWS Summit Japanのデモ展示で使う。観客の注目を集め、高評価をもらえる仕組みを検討・実装する |
| リクエストタイプ | Enhancement（既存機能の拡張・追加） |
| スコープ見積もり | Single Component（会話画面のサジェスト返答機能。フロント中心＋スコアリングエージェントへの軽微な追加） |

### 1.1 背景と経緯

当初は「集客・短時間体験・差別化・運用」の観点で多数の施策候補（ゲストモード、観客向け大型ビュー、感情誇張、称号演出、QR持ち帰り、ライブランキング等）を検討した。議論を通じて、ブースの物理制約・セキュリティ制約・運用方針に照らして候補を絞り込んだ結果、**今回のAI-DLCで新規実装する機能は「サジェスト返答ボタン」1点**に確定した。

絞り込みの過程で除外・代替された項目は「7. スコープ外」に記録する。

## 2. デモ運用の前提条件（確定事項）

| 項目 | 決定 |
|---|---|
| デモのゴール | アセット/ソリューションの認知拡大 |
| 体験形式 | 参加型＋見せる型 |
| 入力方式 | チャット（音声認識は使わない。キーボード打鍵 + サジェスト返答ボタン併用） |
| 物理構成 | 体験PC 1台 → 大型モニターは**ミラーリング接続**（観客向け専用ビューは作らない） |
| 認証 | Cognito認証スキップは禁止。**共通デモユーザー1アカウントを事前ログインして使い回す**（ログイン操作・切り替えは運用で対応、実装不要） |
| リセット運用 | ワンクリックリセットは実装しない。シナリオ再開で代替 |
| ランキング | デモでは使わない |
| 称号・終了演出 | 実装しない |
| QR持ち帰り | 実装しない（ブースパネルにQRコード掲載済み） |
| デモ専用実装 | 行わない。すべて本番でも使える汎用機能として実装する |

## 3. 機能要件

### FR-001: AIによるサジェスト返答候補の生成
- リアルタイムスコアリングエージェント（`cdk/agents/realtime-scoring`）が、スコアリング処理と同時に、ユーザーの次の返答候補を生成する
- 生成する候補は **3件程度** の短文とする
- 候補は、直近のNPC発言と会話の流れを踏まえた、営業担当者（ユーザー）の返答として自然なものとする
- 候補は多様な方向性（例: 強気/共感/質問 など）を持たせ、選択によってメトリクスが変化することを意図する
- 出力は構造化出力モデル `ScoringResult` に `suggestions: List[str]` フィールドを追加する形で返す
- 1件あたりの文字数に上限を設け、レイテンシとトークン消費を抑える

### FR-002: サジェスト返答ボタンのUI表示
- 会話画面（`ConversationPage`）で、NPC応答後にサジェスト返答候補をボタン群として表示する
- 表示位置はメッセージ入力エリア（`MessageInput`）の直上とする
- 各候補はボタンとして表示し、タップ/クリックで選択できる
- 処理中（`isProcessing`）やNPC発話中（`isSpeaking`）は表示を抑制する

### FR-003: サジェスト返答ボタンによる送信
- ユーザーがサジェストボタンを選択すると、その候補テキストを送信メッセージとして会話を送信する
- 既存の送信関数 `sendMessage(inputText?)` を活用し、ボタンから候補テキストを引数で渡して送信する
- フリーテキスト入力欄は引き続き併設し、ユーザーが自由入力することも可能とする（ハイブリッド運用）

### FR-004: シナリオ単位のON/OFF設定
- サジェスト返答ボタン機能は、シナリオ設定で有効/無効を切り替えられる汎用機能とする
- シナリオ設定に `suggestionEnabled: boolean` フラグを追加する
- フラグが無効の場合、候補生成リクエスト/処理を行わず、ボタンも表示しない（不要なトークン消費を回避）
- フラグ未設定時のデフォルトは無効（false）とする

### FR-005: 国際化対応
- サジェスト返答ボタンに関わるUIテキスト（ラベル等）は i18n キー（`conversation.suggestion.*`）で管理し、日本語・英語の両方に対応する
- 生成される返答候補自体は、シナリオの言語設定に従った言語で生成する

## 4. 非機能要件

### NFR-001: パフォーマンス／レイテンシ
- 候補生成は既存のリアルタイムスコアリング呼び出しに相乗りし、**追加のAPI呼び出しを発生させない**
- スコアリングのレイテンシを著しく悪化させないため、候補は短文・件数限定とする

### NFR-002: コスト
- `suggestionEnabled` が無効のシナリオでは候補生成を行わず、トークン消費を発生させない
- 候補の文字数・件数を制限し、トークン消費を抑制する

### NFR-003: 既存機能への影響最小化
- 本機能はフラグ（シナリオ設定 `suggestionEnabled`）で制御し、既存の本番機能・既存シナリオの挙動を変えない
- レスポンス型へのフィールド追加は後方互換とする（`suggestions` 未設定時も既存フローが動作する）
- デモ専用の分岐・別ルートは作らない（汎用機能として既存フローに統合する）

### NFR-004: 型安全性
- TypeScript の型定義を厳密に行い、`any` の使用を避ける
- レスポンス型・シナリオ型に `suggestions` / `suggestionEnabled` を適切に定義する

### NFR-005: アクセシビリティ
- サジェストボタンはキーボード操作可能とし、適切な `aria-label` を付与する（WCAG 2.1 AA準拠の既存方針に従う）

### NFR-006: エラーハンドリング
- 候補生成に失敗した場合でも、スコアリング本来の処理と会話継続に影響を与えない（候補が空でも会話は成立する）
- フロントエンドはサジェストが取得できない場合、ボタンを表示せず通常のフリーテキスト入力にフォールバックする

## 5. 影響範囲（実装観点・参考）

| レイヤー | 対象 | 変更概要 |
|---|---|---|
| バックエンド（エージェント） | `cdk/agents/realtime-scoring/models.py` | `ScoringResult` に `suggestions: List[str]` 追加 |
| バックエンド（エージェント） | `cdk/agents/realtime-scoring/prompts.py` | 候補生成ルールをプロンプトに追記 |
| バックエンド（Lambda） | `cdk/lambda/scenarios/index.py` | create/update のフィールド処理に `suggestionEnabled` 追加 |
| 初期データ | `cdk/data/scenarios.json` | 対象シナリオに `suggestionEnabled` 付与（「【デモ】AWSクラウド移行提案 - 初回商談」等） |
| フロント（型） | `frontend/src/types/api.ts`, `frontend/src/types/index.ts` | `suggestions`（レスポンス）/ `suggestionEnabled`（シナリオ）追加 |
| フロント（サービス） | `frontend/src/services/AgentCoreService.ts`, `ApiService.ts` | `getRealtimeEvaluation` レスポンスに `suggestions` を伝搬 |
| フロント（UI） | `frontend/src/pages/ConversationPage.tsx` + 新規 `SuggestionBar` | 候補のstate管理・ボタン表示・`sendMessage(候補)` 呼び出し |
| i18n | `frontend/src/i18n/locales/{ja,en}.json` | `conversation.suggestion.*` 追加 |

## 6. ブース運用メモ（実装外・参考）

- ブース専用ショートシナリオは新規コードではなく、既存シナリオ「【デモ】AWSクラウド移行提案 - 初回商談」を**通常シナリオの1つ**として短めに決着しやすいようデータ調整（ターン数・初期メトリクス・ゴール条件）して用意する。コード変更は伴わない
- 共通デモユーザー（Cognito）でスタッフが事前ログインしておき、来場者にPCを操作してもらう
- 大型モニターは体験PCのミラーリング出力

## 7. スコープ外（検討の上で除外した項目）

| 項目 | 除外理由 |
|---|---|
| ゲストモード（認証スキップ） | セキュリティ制約により禁止。共通Cognitoユーザーの事前ログインで代替 |
| AgentCoreウォームアップ | コールドスタートは実運用で問題にならない |
| ワンクリックリセット | シナリオ再開で代替可能 |
| 観客向け大型ビュー | 大型モニターはミラーリング接続で対応 |
| 感情表現の強化（誇張） | 今回のスコープから除外 |
| 終了演出・称号判定 | 不要 |
| QR持ち帰り | ブースパネルにQR掲載済み |
| ランキング機能 | デモでは使わない |
| サーバー側セッション同期（2PC連動） | 同一PC・HDMIミラーリングのため不要 |

## 9. 主要要件サマリー

- 今回実装する機能は **サジェスト返答ボタン（AI動的生成）** の1点
- リアルタイムスコアリングエージェントの構造化出力に相乗りして候補を生成（追加API呼び出しなし）
- 会話画面でNPC応答後に候補をボタン表示し、タップで送信。フリーテキスト入力も併設
- シナリオ設定 `suggestionEnabled` でON/OFFできる**本番でも使える汎用機能**として実装（デモ専用実装はしない）
- 日英対応、型安全、アクセシビリティ準拠。既存機能への影響は最小化
